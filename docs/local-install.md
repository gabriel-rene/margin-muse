# Margin Muse on your taskbar

Margin Muse runs as a local server plus an installed PWA: the server starts at
login, and the browser-installed app gives you a dock/taskbar icon that opens
a chromeless window. Notes sync across machines by pointing `MUSE_VAULT_DIR`
at a cloud-synced folder (iCloud/Dropbox/Obsidian vault); the app itself is
installed per machine.

On every machine you need: Node.js 20+, a clone of this repo, `npm install`,
and a `.env.local` (copy `.env.example`; set `MUSE_VAULT_DIR` to your synced
notes folder).

## macOS

```bash
bash scripts/autostart/install-macos.sh
```

Builds the app if needed, installs a `launchd` agent serving
`http://localhost:3323` at login, and prints the PWA install steps
(Chrome/Edge: install icon in the address bar → pins to Dock; Safari:
File → Add to Dock…). Uninstall with `scripts/autostart/uninstall-macos.sh`.
Change the port with `MUSE_PORT=4000 bash scripts/autostart/install-macos.sh`.

## Windows

1. Build once: `npm run build`
2. Create `start-muse.vbs` somewhere permanent (runs the server without a
   console window):

   ```vbs
   Set shell = CreateObject("WScript.Shell")
   shell.CurrentDirectory = "C:\path\to\margin-muse"
   shell.Run "cmd /c npx next start -p 3323", 0
   ```

3. Press `Win+R` → `shell:startup` → drop a shortcut to the `.vbs` there.
4. Open `http://localhost:3323` in Chrome/Edge → **Install** icon in the
   address bar → the app pins to the taskbar like any other program.

## Linux

1. Build once: `npm run build`
2. Create `~/.config/systemd/user/margin-muse.service`:

   ```ini
   [Unit]
   Description=Margin Muse local server

   [Service]
   WorkingDirectory=/path/to/margin-muse
   ExecStart=/usr/bin/env npx next start -p 3323
   Restart=on-failure

   [Install]
   WantedBy=default.target
   ```

3. `systemctl --user enable --now margin-muse`
4. Open `http://localhost:3323` in Chrome/Chromium → Install → the app
   appears in your launcher/taskbar.

## Notes

- The PWA is just a window onto the local server — if the server isn't
  running, the icon opens an error page. The autostart setup above is what
  keeps that from happening.
- No service worker is registered on purpose: the app is server-backed and
  offline caching of a localhost app adds failure modes without benefit.
- Icons are generated by `npm run icons` (`scripts/generate-icons.mjs`,
  dependency-free) and committed.
- A tray/menu-bar Electron shell (server embedded, global shortcut) is the
  planned next step if the PWA setup ever feels too loose.
